Diagrama E - R

El modelo de entidad-relación (MER) de la base de datos ListaDeControl es un modelo conceptual que representa las entidades y relaciones que se encuentran en la base de datos. Las entidades son objetos del mundo real que se representan en la base de datos, como alumnos, cursos, calificaciones, etc. Las relaciones son las conexiones que existen entre las entidades, como la relación entre un alumno y un curso. El MER de la base de datos ListaDeControl consta de las siguientes entidades:

  • Institución: Representa una institución educativa, como una escuela, un colegio o una universidad.
  • TipoUsuario: Representa un tipo de usuario, como alumno, profesor, calificador, etc.
  • Usuario: Representa un usuario individual, como un alumno, un profesor, un calificador, etc.
  • Curso: Representa un curso académico.
  • Localidad: Representa una localidad, como una ciudad, un pueblo o un barrio.
  • Criterios: Representa un criterio de evaluación, como una nota, un examen o un trabajo práctico.
  • Calificador: Representa a una persona que califica a los alumnos.
  • Asignatura: Representa una asignatura.
  • País: Representa un país.

Los resultados de las entrevistas mostraron que los docentes utilizan una lista de alumnos en formato papel para registrar la asistencia y las notas de los estudiantes. Esta lista es elaborada por cada docente y se basa en los criterios de evaluación que considere.

Los docentes expresaron que la lista de papel es una herramienta útil, pero que también tiene algunas limitaciones. En particular, mencionaron que:

  • Es difícil mantener actualizada la lista.
  • Es difícil compartir la lista con otros docentes.
  • Es difícil analizar los datos de la lista.

Las relaciones entre estas entidades son las siguientes:

  • Institución tiene muchos usuarios: Una institución puede tener muchos usuarios, como alumnos, profesores, calificadores, etc.
  • TipoUsuario tiene muchos usuarios: Un tipo de usuario puede tener muchos usuarios, como alumnos, profesores, calificadores, etc.
  • Usuario pertenece a una institución: Un usuario pertenece a una institución educativa, como una escuela, un colegio o una universidad.
  • Usuario tiene un tipo de usuario: Un usuario tiene un tipo de usuario, como alumno, profesor, calificador, etc. Curso tiene muchos alumnos: Un curso puede tener muchos alumnos. Curso tiene una localidad: Un curso está ubicado en una localidad, como una ciudad, un pueblo o un barrio. Criterios tiene muchos cursos: Un criterio de evaluación puede ser utilizado en muchos cursos. Calificador califica a muchos alumnos: Un calificador califica a muchos alumnos. Calificador pertenece a una institución: Un calificador pertenece a una institución educativa, como una escuela, un colegio o una universidad. Asignatura tiene muchos cursos: Una asignatura puede ser impartida en muchos cursos. País tiene muchas instituciones: Un país puede tener muchas instituciones educativas, como escuelas, colegios o universidades. Este MER proporciona una representación conceptual de la base de datos ListaDeControl. Puede utilizarse para comprender cómo se relacionan las entidades y las relaciones en la base de datos.
